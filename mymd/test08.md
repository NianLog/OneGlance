## 编译系统概述

### 编译的流程

```
源代码 → [词法分析] → [语法分析] → [语义分析] → [中间代码生成] → [代码优化] → [目标代码生成] → 目标程序
         ↓            ↓            ↓            ↓                ↓            ↓
        单词序列    语法树      类型检查    三地址码/四元式   优化后的中间代码  汇编/机器码
```

### 解释器与编译器

| 特性 | 编译器 | 解释器 |
|------|--------|--------|
| 执行方式 | 先翻译后执行 | 边翻译边执行 |
| 执行速度 | 快 | 慢 |
| 内存占用 | 较小 | 较大 |
| 跨平台 | 需要重新编译 | 较易 |

<span class="tip-box">Java采用混合方式：先编译为字节码，再由JVM解释执行。</span>

### 编译程序的结构

```

```

## 语法分析

### 上下文无关文法

文法G由四元组组成：G = (VT, VN, S, P)

- VT：终结符集合
- VN：非终结符集合
- S：开始符号
- P：产生式集合

```BNF
<表达式> ::= <项> | <表达式> + <项>
<项>     ::= <因子> | <项> * <因子>
<因子>   ::= ( <表达式> ) | id
```

### 自上而下分析

#### 递归下降分析

```c
void expr() {
    term();
    while (lookahead == '+' || lookahead == '-') {
        match(lookahead);
        term();
    }
}

void term() {
    factor();
    while (lookahead == '*' || lookahead == '/') {
        match(lookahead);
        factor();
    }
}
```

#### 预测分析表

| 非终结符 | + | - | * | / | id | ( | ) | $ |
|---------|---|---|---|---|----|---|---|---|
| E |  E→E+T | E→E-T | - | - | E→T | E→T | - | - |
| T | - | - | T→T*F | T→T/F | T→F | T→F | - | - |

### 自下而上分析

#### 算符优先分析

**优先关系表：**

|  | + | * | ( | ) | id |
|--|---|---|---|---|----|---|
| + | > | < | < | > | < |
| * | > | > | < | > | < |
| ( | < | < | < | = | < |
| ) | > | > | - | > | - |
| id| > | > | - | > | - |

<span class="warning-box">注意：算符优先分析只能处理算符文法，无法处理所有上下文无关文法。</span>

## 中间代码生成

### 三地址码

每条指令最多有三个操作数：

```
t1 := a + b
t2 := t1 * c
t3 := t2 - d
```

### 四元式

| op | arg1 | arg2 | result |
|----|------|------|--------|
| +  | a    | b    | t1     |
| *  | t1   | c    | t2     |
| -  | t2   | d    | t3     |

### 三元式

| 序号 | op | arg1 | arg2 |
|------|-----|------|------|
| (0)  | +  | a    | b    |
| (1)  | *  | (0)  | c    |
| (2)  | -  | (1)  | d    |

<span class="tip-box">中间代码与具体机器无关，便于代码优化和跨平台移植。</span>

## 代码优化

### 优化种类

| 类型 | 说明 | 示例 |
|------|------|------|
| 局部优化 | 基本块内优化 | 常量合并、死代码删除 |
| 循环优化 | 循环内优化 | 强度削弱、代码外提 |
| 全局优化 | 跨基本块优化 | 全局寄存器分配 |

### 常用优化技术

```c
// 优化前
int square(int x) {
    int result = x * x;
    for (int i = 0; i < 10; i++) {
        result = result + 0;
    }
    return result;
}

// 优化后
int square(int x) {
    return x * x;  // 常量折叠、死代码删除
}
```

### DAG优化

有向无环图（Directed Acyclic Graph）用于识别公共子表达式：

```
       *
      / \
     a   b
    / \   \
   *   -   c
  / \       \
 d   e       f

公共子表达式：a*b 和 d*e
```

<span class="tip-box">优化的目标：在不改变程序语义的前提下，提高程序的执行效率。</span>
