## 软件工程概述

### 软件危机的表现

1. 开发成本和进度估计不准确
2. 软件产品质量靠不住
3. 软件通常不可维护
4. 软件没有适当的文档资料
5. 软件成本在计算机系统总成本中所占比例越来越大

<span class="warning-box">软件危机的本质是：软件规模的增长远超开发能力的增长。</span>

### 软件工程的定义

软件工程是应用于软件的定义、开发和维护的一整套方法、技术、工具和管理相结合的、综合性的工程措施。

## 软件过程模型

### 瀑布模型

```
需求分析 → 软件设计 → 程序编码 → 软件测试 → 运行维护
    ↓           ↓           ↓          ↓          ↓
  上游        文档驱动    代码实现   质量检查   持续改进
```

**特点**：阶段式、文档驱动、线性顺序

### 增量模型

将软件产品作为一系列的增量构件来设计、编码、集成和测试。

### 敏捷模型

强调适应性而非预测性，以人为核心而非以过程为核心。

<span class="tip-box">敏捷宣言：个体和交互高于流程和工具；工作的软件高于详尽的文档；客户合作高于合同谈判；响应变化高于遵循计划。</span>

| 原则 | 说明 |
|------|------|
| 快速交付 | 较短的项目迭代周期 |
| 拥抱变化 | 需求变化不可避免 |
| 持续改进 | 团队定期反思如何提高效率 |

## 软件设计

### 设计原则

1. **模块化**：将系统划分为若干模块
2. **抽象化**：抽取出事物的本质特性
3. **信息隐蔽**：每个模块内部包含的信息不需要被其他模块知晓
4. **模块独立**：每个模块尽可能独立完成功能

### 模块耦合类型

| 耦合类型 | 说明 | 优先级 |
|---------|------|-------|
| 非直接耦合 | 模块间没有直接关系 | 最好 |
| 数据耦合 | 通过参数交换数据 | 好 |
| 标记耦合 | 通过参数传递记录 | 一般 |
| 控制耦合 | 传递控制信息 | 较差 |
| 公共耦合 | 共享全局数据 | 差 |
| 内容耦合 | 直接访问模块内部 | 最差 |

### 模块内聚类型

| 内聚类型 | 说明 | 优先级 |
|---------|------|-------|
| 偶然内聚 | 模块内各部分无关联 | 差 |
| 逻辑内聚 | 逻辑相似的功能放一起 | 较差 |
| 瞬时内聚 | 时间相关的功能放一起 | 一般 |
| 过程内聚 | 按特定次序执行的功能 | 较好 |
| 通信内聚 | 使用相同数据的功能 | 较好 |
| 顺序内聚 | 一个功能的输出是另一个功能的输入 | 好 |
| 功能内聚 | 完成单一功能 | 最好 |

<span class="tip-box">设计目标：高内聚、低耦合</span>

## 软件测试

### 测试原则

1. 尽早不断地进行测试
2. 程序员避免测试自己编写的程序
3. 设计测试用例时，包含合理的输入数据和预期的输出结果
4. 充分注意测试中的群集现象
5. 严格执行测试计划，排除测试的随意性
6. 应当对每一个测试结果做全面检查
7. 保存测试计划、测试用例、出错统计和最终分析报告

### 测试方法

#### 黑盒测试

把程序看作一个黑盒子，不考虑程序内部结构，只检查程序是否符合需求规格说明书。

```
输入 ──→ [被测程序] ──→ 输出
           ↓
      需求规格说明书
```

#### 白盒测试

按照程序内部结构，检查程序是否符合设计要求。

```javascript
function add(a, b) {
    return a + b;
}
// 白盒测试需要覆盖所有分支路径
```

### 测试步骤

```
单元测试 → 集成测试 → 确认测试 → 系统测试
   ↓          ↓          ↓          ↓
 模块级      接口级     功能级      性能级
```

## 软件维护

### 维护类型

| 类型 | 说明 | 占比 |
|------|------|------|
| 完善性维护 | 改进和扩充系统功能 | 50% |
| 适应性维护 | 适应硬件和软件环境变化 | 25% |
| 纠错性维护 | 改正在开发期间产生的错误 | 20% |
| 预防性维护 | 防止将来出现问题 | 5% |

### 维护的副作用

1. **代码副作用**：修改代码可能引入新错误
2. **数据副作用**：数据库结构变化可能影响程序
3. **文档副作用**：文档需要与代码保持一致

<span class="warning-box">维护是软件生命周期中最长的阶段，也是最昂贵的阶段。</span>
